SecondStream Explainer

La diferencia entre lo que tenés hoy y lo que buscamos con Agent Harness v1

Hoy SecondStream ayuda a capturar, extraer, revisar y editar información del waste stream. Con Agent Harness v1, SecondStream empieza a mantener una comprensión operativa durable del caso como artifact del producto, no solo como algo que el broker reconstruye en su cabeza.

La frase clave: hoy el sistema guarda datos y sugerencias. Con harness v1, el sistema empieza a guardar y gobernar el entendimiento actual del stream mediante un Discovery Brief versionado, con provenance, conflictos, recomendaciones y memoria reutilizable.

Flujo actual

Lo que ya existe hoy en el código

1. Entra información

Discovery wizard, bulk import, archivos, quick capture, texto, audio, voice interview.

↓
2. AI extrae o sugiere

Drafts, transcript, questionnaire suggestions, file insights, candidate fields.

↓
3. El humano revisa piezas sueltas

Acepta/rechaza drafts, corrige campos, actualiza workspace, revisa suggestions.

↓
4. El workspace guarda verdad

Base fields, questionnaire, context note, custom fields, follow-up state.

↓
5. El broker arma el entendimiento real en su cabeza

El sistema ayuda con extracción y review, pero todavía no sostiene una comprensión operativa unificada del caso.

Flujo con harness v1

Lo nuevo que agrega el primer agente

1. Entra información

Igual que hoy: documentos, texto, audio, workspace changes, reviewed drafts.

↓
2. Corre el Discovery Completion Agent

Lee el stream actual, client/location, historial del stream, dossiers similares y org knowledge aprobado.

↓
3. Genera un Discovery Brief

Un artifact único con: qué sabemos, qué falta, blockers, conflictos, confianza, provenance y next-best actions.

↓
4. El broker revisa un entendimiento unificado

No revisa solo piezas sueltas. Revisa la comprensión actual del caso como objeto del producto.

↓
5. El sistema acumula memoria

Session summaries, observations, historical dossiers, promoted org knowledge y review candidates.
