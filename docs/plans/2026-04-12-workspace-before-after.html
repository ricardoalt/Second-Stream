  • + Discovery Brief: comprension unificada del caso como artifact
  • + Point types: Fact, Assumption, Conflict, Recommendation, Question
  • + Provenance: cada punto dice de donde salio
  • + Workspace Suggestions: el brief produce sugerencias revisables, no escribe directo
  • + Versioned: cada refresh crea una version, con diff
  • + Agent Runs: historial de ejecuciones visibles
  • + Memory: observations, dossiers historicos, org knowledge
  • + Broker control: correccion punto por punto, approved/deferred/rejected
La frase mas simple posible:
Hoy el sistema guarda datos del caso (campos, archivos, suggestions).
Con harness, el sistema empieza a guardar y gobernar el entendimiento del caso como un artifact versionado, con fuentes, conflictos, recomendaciones y memoria durable.
